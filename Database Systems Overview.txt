<---------------Quiz - Database Introduction-------------------->
Q1) Answer:
b) Collection of programs to access data
A DBMS includes both the interrelated data and the programs used to access it.

Q2) Answer: b) Database application and Database
DBMS acts as a middleware between the application and the actual database.

Q3) Answer:
a) Data redundancy and inconsistency
b) Difficulty in fetching data
c) Data isolation is not present
File systems suffer from redundancy, difficulty in querying data, and poor data isolation. High cost is a DBMS concern, not file systems.

Q4) Answer:
a) Maintain accuracy of data
b) Utilities for performance analysis and logging
c) Ensure modifications are successful or not done at all
d) Recovery mechanism for data
DBMS provides data accuracy, logging, atomicity (all-or-nothing transactions), and crash recovery.

Q5) Answer: c) Data can be accessed by multiple programs
DBMS supports concurrent shared access, unlike file systems which struggle with multi-user scenarios.

Q6) Answer:
a) Protect data from system crash
c) Authorized access
DBMS provides crash protection (durability) and access control (authorization). It is not necessarily low cost, and it actually reduces application dependence on storage.

<-------------------- Quiz - Data Integrity-------------------------->


Q1. The relationship between the two relations is created using __________
Answer: c) Foreign key

Q2. Which statements are FALSE about Primary key?
Answer:

b) There can be more than one Primary key for a table.
d) Columns with string (Char/Varchar2) data types cannot be made Primary key because they are large.


Q3. Suggest the best Primary key for Customer entity.
Answer: a) CustId
CustId is preferred as it is numeric, short, and does not change over time unlike Email or ContactNo.

Q4. Identify the Candidate key for the RechargePlan table.
Answer: d) Operator, Plans

Q5. Identify the Primary key for the RechargePayment table.
Answer: c) RechargeId

Q6. Identify the Foreign key for the RechargePayment table.
Answer: d) Operator, Plans
Operator and Plans together reference the RechargePlan table's candidate key, making them the Foreign key in RechargePayment.

Q2. In a many to one relationship, the primary key acts as foreign key on which side?
Answer: b) On the side where many relationship is defined

Q3. How many tables does a many to many relationship result in?
Answer: b) Three
A junction/bridge table is created in addition to the two entity tables to resolve the many-to-many relationship.
